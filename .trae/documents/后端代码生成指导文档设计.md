# 后端代码生成指导文档设计

## 文档结构

### 1. 第一部分：项目总览、项目结构、核心功能模块
- 项目背景和目标
- 技术栈
- 核心功能模块
- 项目目录结构

### 2. 第二部分：模块关联索引（阶段1）
- 阶段1：系统地基期
- 模块依赖关系
- 模块生成顺序

### 3. 第三部分：模块关联索引（阶段2-3）
- 阶段2：AI融合期
- 阶段3：认知辅助成型期
- 模块依赖关系
- 模块生成顺序

### 4. 第四部分：代码生成指南、进度跟踪机制、测试生成策略
- 代码生成流程
- 模块生成步骤
- 代码风格和规范
- 进度跟踪文件格式
- 生成日志格式
- 断点恢复机制
- 测试生成策略
- 上下文中断处理方案

## 实现步骤

1. **创建代码生成指导文档第一部分**：项目总览和核心功能模块
2. **创建代码生成指导文档第二部分**：阶段1模块关联索引
3. **创建代码生成指导文档第三部分**：阶段2-3模块关联索引
4. **创建代码生成指导文档第四部分**：代码生成指南和进度跟踪机制
5. **创建进度跟踪文件**：用于跟踪代码生成进度
6. **创建示例生成日志**：用于演示生成日志格式

## 核心功能

- **模块关联索引**：建立详细的模块依赖关系，确保代码生成顺序正确
- **进度跟踪机制**：实现进度跟踪文件和生成日志，便于AI跟踪生成进度
- **断点恢复机制**：设计断点恢复方案，解决上下文中断问题
- **测试生成策略**：制定测试生成计划，确保代码质量
- **上下文中断处理**：提供上下文中断处理方案，确保生成过程顺利进行

## 技术要点

- 采用与前端类似的文档结构，便于AI理解和使用
- 基于Clean Architecture设计理念，确保代码生成符合架构规范
- 严格遵循TypeScript和Fastify最佳实践
- 实现完整的测试生成策略，确保代码质量
- 设计有效的进度跟踪机制，解决上下文中断问题
