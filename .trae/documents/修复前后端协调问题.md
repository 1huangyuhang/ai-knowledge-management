# 修复前后端协调问题

## 问题清单

1. **API路径缺少完整版本号前缀**：所有API端点路径只显示相对路径，没有完整的`/api/v1`前缀
2. **模块编号错误**：文件处理模块和语音处理模块编号重复（都为3.2），且文件处理模块的接口编号错误
3. **缺少API模块**：推送通知模块、iOS特定功能模块和健康检查模块在API规范文档中未定义
4. **接口命名不一致**：部分接口参数和响应字段命名不一致
5. **缺少统一的错误码定义**：虽然有错误响应格式，但没有完整的错误码列表

## 修复方案

### 1. 更新API规范文档

- **修复所有API路径**：为每个API端点添加完整的`/api/v1`前缀
- **修正模块编号**：重新编号所有模块，确保编号唯一且连续
  - 认证模块：3.1 → 保持不变
  - 文件处理模块：3.2 → 保持不变，修正接口编号为3.2.1, 3.2.2, 3.2.3
  - 语音处理模块：3.2 → 改为3.3，接口编号改为3.3.1, 3.3.2, 3.3.3
  - AI调度模块：3.3 → 改为3.4，接口编号改为3.4.1, 3.4.2, 3.4.3
  - 输入整合模块：3.4 → 改为3.5，接口编号改为3.5.1, 3.5.2
- **添加缺失的API模块**：
  - 推送通知模块：3.6，包含设备令牌管理和推送通知发送接口
  - iOS特定功能模块：3.7，包含iOS客户端特定功能接口
  - 健康检查模块：3.8，包含服务健康检查接口
- **统一接口命名**：确保所有接口参数和响应字段使用一致的命名规则
- **添加统一的错误码定义**：提供完整的错误码列表和说明

### 2. 验证所有文档一致性

- 确保API设计文档和API规范文档中的模块划分一致
- 确保WebSocket事件名称在所有文档中一致
- 确保数据模型定义在所有文档中一致
- 确保认证机制在所有文档中一致

### 3. 更新前端集成设计文档

- 确保前端API调用示例使用完整的API路径
- 确保前端响应处理逻辑与统一的响应格式一致
- 确保前端WebSocket事件处理与后端一致

## 修复步骤

1. 首先更新API规范文档，修复所有API路径和模块编号
2. 在API规范文档中添加缺失的API模块
3. 在API规范文档中添加统一的错误码定义
4. 验证API设计文档和API规范文档的一致性
5. 更新前端集成设计文档，确保与后端一致
6. 验证所有文档的最终一致性

- 所有API端点路径都包含完整的`/api/v1`前缀
- 所有模块编号唯一且连续
- 所有API模块都在API规范文档中定义
- 所有接口参数和响应字段命名一致
- 提供完整的错误码列表和说明
- 前后端文档完全一致，确保集成顺利
