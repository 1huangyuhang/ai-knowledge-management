# 前端页面设计规范技术文档

## 1. 文档概述

### 1.1 项目背景
基于AI认知辅助系统后端，设计开发一款iOS平台的AI Voice Interaction App，实现用户与AI系统的语音交互，帮助用户管理和优化认知结构。

### 1.2 设计目标
- 实现与后端API的无缝集成
- 提供直观、流畅的语音交互体验
- 支持认知模型的可视化展示
- 实现多维度分析结果的可视化
- 支持个性化定制功能
- 确保应用性能和安全性

## 2. 技术栈

### 2.1 核心技术
- **开发语言**：Swift 5.9+
- **UI框架**：SwiftUI 5.0+
- **语音识别**：Speech Framework
- **网络请求**：URLSession + Async/Await
- **状态管理**：SwiftUI ObservableObject + Environment
- **WebSocket**：URLSessionWebSocketTask
- **数据持久化**：Core Data

### 2.2 第三方依赖
| 依赖 | 用途 |
|------|------|
| Alamofire | 网络请求封装 |
| Combine | 响应式编程 |
| SwiftyJSON | JSON解析 |
| SDWebImageSwiftUI | 图片加载与缓存 |
| SwiftLint | 代码质量检查 |

## 3. 架构设计

### 3.1 分层架构
```
┌─────────────────┐
│ Presentation   │  # SwiftUI视图层
├─────────────────┤
│ ViewModel      │  # 业务逻辑与状态管理
├─────────────────┤
│ Service        │  # API服务与数据处理
├─────────────────┤
│ Model          │  # 数据模型
└─────────────────┘
```

### 3.2 与后端架构对应关系
| 后端分层 | 前端对应层 |
|----------|------------|
| Presentation（API层） | Service层 |
| Application层 | ViewModel层 |
| Domain层 | Model层 |
| Infrastructure层 | Service层 |
| AI Capability层 | Service层 |

## 4. 页面设计规范

### 4.1 页面结构

#### 4.1.1 主页面结构
- **顶部导航栏**：应用标题、设置按钮
- **中间内容区**：核心功能展示
- **底部操作栏**：语音输入按钮、主要功能入口

#### 4.1.2 核心页面列表
1. **欢迎页面**
2. **登录/注册页面**
3. **主页（认知模型概览）**
4. **语音交互页面**
5. **多维度分析页面**
6. **认知模型详情页面**
7. **个性化设置页面**
8. **历史记录页面**

### 4.2 组件设计规范

#### 4.2.1 语音输入组件
- **样式**：圆形浮动按钮，位于屏幕底部中央
- **状态**：默认、录音中、录音完成
- **交互**：长按录音，松开停止
- **反馈**：录音时长显示、波形动画

#### 4.2.2 认知模型可视化组件
- **类型**：力导向图、层次图、雷达图
- **交互**：缩放、拖拽、点击详情
- **样式**：节点颜色区分概念类型，边粗细表示关系强度

#### 4.2.3 分析结果展示组件
- **类型**：雷达图、柱状图、饼图
- **数据更新**：实时更新，平滑过渡动画
- **交互**：点击查看详情，切换分析维度

### 4.3 设计系统

#### 4.3.1 色彩规范
| 颜色 | 用途 |
|------|------|
| 主色调（#4A90E2） | 按钮、导航栏、强调元素 |
| 辅助色（#50E3C2） | 成功状态、积极反馈 |
| 警告色（#F5A623） | 警告状态、需要注意的元素 |
| 错误色（#D0021B） | 错误状态、负面反馈 |
| 背景色（#F8F8F8） | 页面背景 |
| 卡片色（#FFFFFF） | 卡片、容器背景 |

#### 4.3.2 字体规范
| 字号 | 用途 |
|------|------|
| 24pt | 页面标题 |
| 20pt | 卡片标题 |
| 16pt | 正文内容 |
| 14pt | 辅助文字 |
| 12pt | 提示文字 |

#### 4.3.3 间距规范
| 间距 | 用途 |
|------|------|
| 8pt | 小间距（按钮内边距、文字间距） |
| 16pt | 中间距（卡片间距、区域间距） |
| 24pt | 大间距（页面边距、模块间距） |
| 32pt | 超大间距（主要内容区域） |

## 5. API集成规范

### 5.1 API请求规范
- **Base URL**：从配置文件读取
- **认证**：JWT Token，存储在Keychain
- **请求格式**：JSON
- **响应格式**：统一格式，包含success、data、error字段
- **错误处理**：统一的错误码和错误信息

### 5.2 核心API集成

#### 5.2.1 认证相关API
- 登录
- 注册
- 刷新Token

#### 5.2.2 认知模型相关API
- 获取认知模型列表
- 获取认知模型详情
- 更新认知模型
- 删除认知模型

#### 5.2.3 语音交互相关API
- 语音转文本
- 文本转语音
- AI对话生成

#### 5.2.4 分析相关API
- 获取多维度分析结果
- 触发思维类型分析
- 获取分析历史

#### 5.2.5 个性化相关API
- 获取个性化配置
- 更新个性化配置
- 恢复默认配置

### 5.3 WebSocket集成
- **连接管理**：自动重连机制
- **事件处理**：认知模型更新、分析结果推送
- **心跳机制**：定期发送心跳包，保持连接

## 7. 状态管理

### 7.1 全局状态
- **用户状态**：登录状态、用户信息
- **网络状态**：在线/离线状态
- **WebSocket连接状态**：连接中/已连接/已断开

### 7.2 页面状态
- **加载状态**：加载中/加载完成/加载失败
- **空状态**：无数据提示
- **错误状态**：错误信息展示

### 7.3 状态更新机制
- 使用SwiftUI的@ObservableObject和@EnvironmentObject
- 支持异步更新和状态持久化
- 实现状态变更通知机制

## 8. 安全规范

### 8.1 数据安全
- **敏感数据加密**：用户凭证存储在Keychain
- **网络请求加密**：使用HTTPS
- **数据传输加密**：敏感数据加密传输

### 8.2 权限管理
- **语音权限**：明确的权限申请提示
- **数据访问权限**：最小权限原则
- **隐私政策**：清晰的隐私政策展示

### 8.3 输入验证
- **客户端验证**：输入数据格式验证
- **服务器验证**：依赖服务器最终验证
- **防止SQL注入**：参数化查询

## 9. 性能优化

### 9.1 网络优化
- **请求合并**：减少网络请求次数
- **缓存策略**：合理的缓存机制
- **延迟加载**：按需加载数据

### 9.2 UI优化
- **列表优化**：使用LazyVStack/LazyHStack
- **图片优化**：异步加载、缓存、压缩
- **动画优化**：使用SwiftUI内置动画，避免过度动画

### 9.3 内存优化
- **对象复用**：重用UI组件
- **内存监控**：定期检查内存使用情况
- **及时释放资源**：避免内存泄漏
