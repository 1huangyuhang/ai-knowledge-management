## 1. 代码生成指南

### 1.1 代码生成流程

#### 1.1.1 准备阶段
1. **检查依赖**：确认当前模块的所有依赖已生成完成
2. **阅读相关文档**：仔细阅读当前模块的技术实现文档和相关核心文档
3. **检查模块状态**：查看进度跟踪文件，确认模块的当前状态
4. **准备生成环境**：确保开发环境配置正确

#### 1.1.2 生成阶段
1. **生成代码文件**：根据技术实现文档生成代码文件
2. **遵循代码规范**：严格按照项目的代码风格指南生成代码
3. **实现所有功能点**：确保实现技术文档中描述的所有功能
4. **添加必要注释**：为核心逻辑和复杂代码添加注释
5. **处理错误情况**：实现适当的错误处理和边界情况处理

#### 1.1.3 验证阶段
1. **检查代码完整性**：确保所有功能点已实现
2. **运行编译检查**：确保代码可以成功编译
3. **运行静态分析**：检查代码是否符合静态分析规则
4. **生成测试代码**：根据模块功能生成相应的测试代码
5. **运行测试**：确保测试通过

### 1.3 代码风格和规范

#### 1.3.1 Swift代码规范
- 遵循Swift API设计指南
- 使用SwiftUI进行UI开发
- 采用Combine进行异步编程
- 遵循MVC或MVVM架构
- 使用强类型，避免Any类型
- 实现适当的错误处理

#### 1.3.2 命名规范
- 类名：大驼峰命名法（如`APIService`）
- 方法名：小驼峰命名法（如`request`）
- 属性名：小驼峰命名法（如`accessToken`）
- 枚举名：大驼峰命名法（如`APIEndpoint`）
- 常量名：大写蛇形命名法（如`MAX_RETRY_COUNT`）

#### 1.3.3 注释规范
- 为类和方法添加文档注释
- 为复杂逻辑添加单行注释
- 为关键算法添加详细注释
- 使用Markdown格式编写文档注释

3. **确定恢复点**：根据最后生成的文件和模块状态确定恢复点
4. **恢复生成**：从恢复点继续生成剩余代码
5. **更新进度**：恢复生成后更新进度跟踪文件

## 3. 测试生成策略

### 3.1 测试类型和覆盖范围

#### 3.1.1 测试类型
- **单元测试**：测试单个函数或类的功能
- **集成测试**：测试模块之间的交互
- **UI测试**：测试用户界面的功能和交互
- **性能测试**：测试应用的性能表现

#### 3.1.2 测试覆盖范围
- **核心功能**：所有核心功能模块必须有测试覆盖
- **边界情况**：测试边界条件和异常情况
- **错误处理**：测试错误处理逻辑
- **性能**：测试关键功能的性能

### 3.2 测试生成步骤

#### 3.2.1 单元测试生成步骤
1. **确定测试目标**：根据模块功能确定需要测试的类和函数
2. **创建测试文件**：在项目的Tests目录下创建测试文件
3. **编写测试用例**：
   - 测试正常功能
   - 测试边界情况
   - 测试错误处理
4. **运行测试**：确保测试通过
5. **更新测试**：当模块功能变更时更新测试

#### 3.2.2 集成测试生成步骤
1. **确定集成点**：确定模块之间的集成点
2. **创建集成测试文件**：在项目的Tests目录下创建集成测试文件
3. **编写集成测试用例**：测试模块之间的交互
4. **运行集成测试**：确保集成测试通过
5. **更新集成测试**：当模块交互变更时更新集成测试

### 3.3 测试框架和工具

#### 3.3.1 测试框架
- **XCTest**：用于单元测试和UI测试
- **CombineTestExtensions**：用于测试Combine发布者
- **MockingBird**：用于创建模拟对象

#### 3.3.2 测试工具
- **Xcode Test Navigator**：用于管理和运行测试
- **XCTestHTMLReport**：用于生成测试报告
- **SwiftLint**：用于检查代码质量

### 3.4 测试执行和报告

#### 3.4.1 测试执行
1. **单个测试**：通过Xcode Test Navigator运行单个测试
2. **测试套件**：运行整个测试套件

3. **修复失败的测试**：对失败的测试进行修复

## 4. 代码生成最佳实践

### 4.1 提高生成效率的方法
1. **批量生成**：对于依赖关系紧密的模块，可以批量生成
2. **模块化生成**：按照模块的功能边界生成代码
3. **复用代码**：对于相似功能，复用已生成的代码
4. **并行生成**：对于独立模块，可以并行生成

### 4.2 确保代码质量的方法
1. **遵循代码规范**：严格按照项目的代码风格指南生成代码
2. **添加注释**：为核心逻辑和复杂代码添加注释
3. **生成测试**：为每个模块生成相应的测试
4. **运行静态分析**：使用SwiftLint等工具检查代码质量
5. **运行编译检查**：确保代码可以成功编译

### 4.3 处理复杂功能的方法
1. **分解功能**：将复杂功能分解为多个简单功能
2. **分层实现**：按照分层架构实现功能
3. **逐步迭代**：逐步实现功能，不断测试和优化
4. **参考相关文档**：参考核心文档和技术实现文档

## 5. 上下文中断处理方案

### 5.1 预防上下文中断的方法
1. **限制生成范围**：每次生成的代码量不要过大
2. **频繁更新进度**：定期更新进度跟踪文件
3. **生成小块代码**：将代码分解为小块生成
4. **保存中间状态**：定期保存生成的中间状态

### 5.2 处理上下文中断的方法
1. **检查进度跟踪文件**：了解最后生成的状态
2. **检查已生成的文件**：确认已生成的代码
3. **确定恢复点**：根据最后生成的状态确定恢复点
4. **恢复生成**：从恢复点继续生成
5. **验证生成结果**：恢复生成后验证代码的完整性
