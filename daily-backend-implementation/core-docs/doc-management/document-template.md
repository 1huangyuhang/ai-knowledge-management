## 4. 文档命名规范

### 4.1 命名原则

- **清晰性**：文件名应清晰反映文档的内容和用途
- **一致性**：遵循项目的命名规范，保持命名的一致性
- **简洁性**：文件名应简洁明了，避免过长
- **可搜索性**：使用易于搜索的关键词

### 4.2 命名格式

#### 4.2.1 技术实现文档

- 对于包含技术实现细节的文档，使用以下命名格式：
  ```
  [编号]-[功能描述]-technical-implementation.md
  ```
  - 示例：`61-suggestion-logic-technical-implementation.md`

#### 4.2.2 设计文档

- 对于设计类文档，使用以下命名格式：
  ```
  [编号]-[功能描述].md
  ```
  - 示例：`01-project-overview.md`

#### 4.2.3 报告文档

- 对于报告类文档，使用以下命名格式：
  ```
  [编号]-[报告类型]-[主题].md
  ```
  - 示例：`99-database-design-evaluation-report.md`

### 4.3 注意事项

- 文件名使用小写字母，单词之间用连字符(-)分隔
- 避免使用特殊字符和空格
- 保持编号的连续性和一致性
- 新文档应遵循现有命名模式，不再引入新的命名格式
