# 技术实现白皮书（Technical Design Document）

> 本文档是**纯技术层面的工程文档**，用于：
>
> * 指导具体代码实现
> * 作为 AI 生成代码、模块、接口时的技术依据
> * 明确每一层“用什么技术、为什么、如何替换”

---

# 1. 技术设计总原则（不可违背）

## 1.1 核心原则

1. **Domain First**：认知模型优先于技术选型
2. **AI is a Dependency, not the Core**：AI 是外部能力，不是系统中心
3. **Replaceable by Design**：任何 AI / DB / 框架都可替换
4. **Single Responsibility**：每个模块只解决一个问题

---

## 1.2 技术约束

* 不使用前端复杂框架（初期）
* 不引入微服务
* 不使用魔法式 Agent 框架

目标：**清晰 > 智能 > 炫技**

---

# 2. 技术栈总览（最终态）

## 2.1 后端核心

| 维度  | 技术             | 选择原因        |
| --- | -------------- | ----------- |
| 语言  | TypeScript     | 强类型 + AI 友好 |
| 运行时 | Node.js 18+    | 生态成熟        |
| 框架  | Fastify        | 性能 + 简洁     |
| 构建  | ts-node / tsup | 新手友好        |

---

## 2.2 AI 技术栈

| 能力        | 技术                 | 说明      |
| --------- | ------------------ | ------- |
| LLM       | 云端 LLM API         | 用于解析与生成 |
| Embedding | text-embedding     | 用于认知相似度 |
| Prompt    | JSON Schema Prompt | 强结构输出   |

> 约束：**AI 只能输出 Proposal，不得直接写数据库**

---

## 2.3 数据与存储

| 类型    | 技术      | 用途   |
| ----- | ------- | ---- |
| 结构化数据 | SQLite  | 核心业务 |
| 向量数据  | Qdrant  | 语义相似 |
| 关系数据  | Graph 表 | 认知关系 |

---

# 3. 分层技术实现细则

## 3.1 Domain Layer（纯净层）

**技术要求**：

* 不依赖任何第三方库
* 不依赖数据库
* 不依赖 AI

### 文件结构

```
src/domain/
├── entities/
│   ├── CognitiveConcept.ts
│   ├── CognitiveRelation.ts
│   └── UserCognitiveModel.ts
├── services/
│   └── CognitiveModelService.ts
└── value-objects/
```

### 技术要点

* 使用不可变对象（尽量）
* 所有规则写成方法
* 不允许出现 HTTP / SQL / Prompt

---

## 3.2 Application Layer（用例层）

**职责**：

* 编排流程
* 调用 Domain
* 不处理技术细节

### 文件结构

```
src/application/
├── usecases/
│   └── IngestThoughtUseCase.ts
├── ports/
│   ├── ThoughtRepository.ts
│   └── CognitiveModelRepository.ts
└── dtos/
```

### 技术要点

* 依赖倒置（只依赖接口）
* 不 new 具体实现

---

## 3.3 Infrastructure Layer（技术实现层）

### 数据存储

```
src/infrastructure/persistence/
├── sqlite/
│   ├── ThoughtRepositoryImpl.ts
│   └── CognitiveModelRepositoryImpl.ts
```

* 使用简单 SQL
* 不做 ORM 魔法

---

### AI 能力实现

```
src/infrastructure/ai/
├── LLMClient.ts
├── PromptTemplates.ts
├── CognitiveParser.ts
└── EmbeddingService.ts
```

#### 技术规范

* Prompt 必须版本化
* 输出必须校验 JSON Schema
* 失败要可恢复

---

## 3.4 Interfaces Layer（交互层）

```
src/interfaces/http/
├── routes/
│   ├── thought.ts
│   └── model.ts
└── server.ts
```

* HTTP 只做参数校验
* 不写业务逻辑

---

# 4. AI 与系统的交互协议（核心）

## 4.1 Proposal 机制

AI **只能**返回：

```
CognitiveProposal {
  concepts: ConceptCandidate[]
  relations: RelationCandidate[]
  confidence: number
}
```

是否采纳：

* Domain Service 决定

---

## 4.2 Prompt 工程规范

* System Prompt 固定
* User Prompt 最小化
* 示例必须存在

---
