# Day 04: 画对象关系图

## 当日主题

绘制 Domain 对象之间的关系图，明确实体间的依赖和关联。

## 技术要点

- 对象关系图设计
- 实体关联类型
- 聚合关系
- 领域模型可视化
- 关系图绘制工具

## 开发任务

1. 分析 Domain 对象间的关系：
   - 识别一对一、一对多、多对多关系
   - 确定聚合根
   - 定义关联规则

2. 绘制对象关系图：
   - 使用合适的工具（如 Draw.io、Mermaid 等）
   - 清晰展示对象间的关系
   - 标注关系类型（如聚合、组合、关联）
   - 展示对象的核心属性

3. 验证关系图的合理性：
   - 检查是否符合业务逻辑
   - 确保关系定义清晰
   - 确认聚合根的选择是否正确

4. 优化关系设计：
   - 简化不必要的关系
   - 确保关系方向正确
   - 考虑性能影响

## 验收标准

- 对象关系图完整，包含所有核心 Domain 对象
- 关系类型标注清晰
- 聚合根明确
- 关系设计符合业务逻辑
- 关系图易于理解

## 交付物

- 对象关系图（电子版本）
- 关系定义文档
- 聚合根设计说明
