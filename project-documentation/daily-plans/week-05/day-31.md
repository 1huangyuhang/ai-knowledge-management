# Day 31: 接入 LLM（解析 Thought 第一部分）

## 当日主题

开始接入 LLM 服务，实现思维片段的认知解析功能，将第一阶段的 mock AI 替换为真实的 LLM 服务。

## 技术要点

- LLM API 集成
- API 密钥管理
- Prompt 工程
- 结构化输出解析
- 错误处理与重试机制
- 异步编程
- 日志记录

## 开发任务

1. 准备 LLM 开发环境：
   - 选择 LLM 提供商（如 OpenAI、Anthropic、Google 等）
   - 注册 API 密钥
   - 安装必要的 SDK 或客户端库
   - 配置环境变量管理（如 dotenv）

2. 设计 LLM 客户端接口实现：
   - 实现 `LLMClient` 接口的具体实现
   - 支持不同 LLM 提供商的适配
   - 实现请求重试机制
   - 实现错误处理
   - 实现请求超时处理

3. 设计和优化 Prompt：
   - 设计用于认知解析的 Prompt
   - 确保 Prompt 能生成结构化的输出
   - 测试 Prompt 的效果
   - 优化 Prompt 以提高解析质量

4. 实现结构化输出解析：
   - 使用 JSON Schema 验证 LLM 输出
   - 实现输出格式转换
   - 处理解析错误
   - 实现 fallback 机制

5. 集成 LLM 客户端到系统：
   - 配置依赖注入
   - 替换原有的 mock 实现
   - 确保 Application 层能正确调用 LLM 服务

6. 编写 LLM 相关测试：
   - 测试 LLM 客户端的基本功能
   - 测试错误处理和重试机制
   - 测试结构化输出解析
   - 测试不同输入场景

7. 实现 LLM 调用日志：
   - 记录 LLM 调用的详细信息
   - 记录请求和响应
   - 记录调用时间和成本
   - 与系统日志系统集成

## 验收标准

- LLM 开发环境配置正确
- LLM 客户端接口实现完成
- Prompt 设计合理，能生成结构化输出
- 结构化输出解析工作正常
- LLM 客户端能正确集成到系统中
- 测试用例通过
- 日志记录完整
